## 1. サービス概要

### サービス名

**Lオペ for CLINIC**（クリニック特化 LINE 運用プラットフォーム）

### コンセプト

LINE公式アカウントを「クリニックの業務基盤」へ進化させる、医療機関専用のオールインワン運用プラットフォームです。

患者の問診・予約・決済・配送・再処方の全フローをLINE上で完結させ、管理者向けにはトーク・友だち管理・タグ・リッチメニュー・ステップ配信・ワークフロー自動化・分析機能を提供します。
AIによる音声カルテ自動生成・医学用語補正・自動返信・文章補助機能も搭載し、診察業務の効率化を支援します。
電子カルテ（EHR）との双方向連携にも対応し、既存システムとのデータ統合が可能です。

### 技術基盤

| 要素 | 概要 |
|------|------|
| アプリケーション | Webアプリケーション（レスポンシブ対応） |
| インフラ | クラウドホスティング（国内リージョン） |
| データ保護 | AES-256暗号化、SSL/TLS通信、RLS（行レベルセキュリティ） |
| マルチテナント | クリニックごとに独立したデータ領域（サブドメイン方式） |
| 外部連携 | LINE Messaging API / 決済サービス（3社） / 音声認識AI / LLM / Google Calendar / EHR |

## 4. 主要業務フロー

### 4.1 患者登録→購入

```
LINE友だち追加 → 個人情報入力 → 電話番号認証 → 商品選択 → 決済（クレジット/銀行振込）→ 購入完了 → 配送
```

### 4.2 問診→診察→処方

```
初回問診入力 → 診察予約 → リマインダー自動送信 → 来院・診察 → カルテ記入（SOAP/音声入力） → 処方判定
```

### 4.3 再処方

```
再処方問診回答 → 申請 → 管理者確認 → 医師が承認/却下 → 承認時: カルテ自動生成→決済→配送
```

### 4.4 AIカルテ生成

```
音声録音（最大5分）→ 文字起こし（Deepgram/Groqフォールバック）→ 医学用語補正（Claude）→ SOAP形式カルテ自動生成 → 医師が確認・編集 → 保存
```

### 4.5 配送

```
決済完了 → 発送待ち一覧 → CSV出力（ヤマト/ゆうパック）→ 追跡番号登録 → LINE発送通知 → 配送状況確認
```

### 4.6 ステップ配信

```
トリガー発火（友だち追加/タグ/キーワード/フォーム回答/手動）→ シナリオにエンロール → 遅延判定 → メッセージ送信 → 次ステップ or 完了
```

### 4.7 ワークフロー自動化

```
トリガー（友だち追加/タグ追加/キーワード等）→ 条件分岐 → アクション実行（タグ付与/メニュー切替/メッセージ送信/API呼び出し）
```

## 5. セキュリティ

| 対策 | 概要 |
|------|------|
| 通信暗号化 | SSL/TLS（HTTPS）による全通信の暗号化 |
| データ暗号化 | AES-256-GCM による設定情報の暗号化保存 |
| 行レベルセキュリティ | テナントごとのデータ分離（RLS全テーブル） |
| 2要素認証 | TOTP + バックアップコードによる2FA |
| CSRF保護 | Double Submit Cookieパターン |
| レート制限 | メール単位・IP単位のログイン試行回数制限（Upstash Redis） |
| CSP | Content Security Policy による XSS 対策 |
| 入力検証 | Zodスキーマによる全API入力バリデーション（36スキーマ） |
| 監査ログ | 管理操作の全履歴記録（3年保存+5年自動削除） |
| セッション管理 | サーバー側セッション管理（24時間有効期限・同時3セッション制限） |
| パスワードポリシー | 8文字以上・大小英数記号必須・90日有効期限・過去5回再利用禁止 |
| ログインアラート | 新しいIPからのログイン時にメール通知 |
| Webhook署名検証 | LINE・Square・GMOからのWebhookの署名検証 |
| 分散ロック | Redis SET NXによるCronジョブの排他制御 |
| 冪等性保証 | Webhook・リマインダー等の重複処理防止 |
