## 1. サービス概要

### サービス名

**Lオペ for CLINIC**（クリニック特化 LINE 運用プラットフォーム）

### コンセプト

LINE公式アカウントを「クリニックの業務基盤」へ進化させる、医療機関専用のオールインワン運用プラットフォームです。

患者の問診・予約・決済・配送・再処方の全フローをLINE上で完結させ、管理者向けにはトーク・友だち管理・タグ・リッチメニュー・ステップ配信・分析機能を提供します。
AIによる音声カルテ自動生成・医学用語補正・自動返信機能も搭載し、診察業務の効率化を支援します。

### 技術基盤

| 要素 | 概要 |
|------|------|
| アプリケーション | Webアプリケーション（レスポンシブ対応） |
| インフラ | クラウドホスティング（国内リージョン） |
| データ保護 | AES-256暗号化、SSL/TLS通信、RLS（行レベルセキュリティ） |
| マルチテナント | クリニックごとに独立したデータ領域 |
| 外部連携 | LINE Messaging API / 決済サービス / 音声認識AI / LLM |

## 4. 主要業務フロー

### 4.1 患者登録→購入

```
LINE友だち追加 → 個人情報入力 → 商品選択 → 決済（クレジット/銀行振込）→ 購入完了 → 配送
```

### 4.2 問診→診察→処方

```
初回問診入力 → 診察予約 → リマインダー自動送信 → 来院・診察 → カルテ記入 → 処方判定
```

### 4.3 再処方

```
再処方問診回答 → 申請 → 医師が確認 → 承認/却下 → 承認時: 決済→配送
```

### 4.4 AIカルテ生成

```
音声録音（最大5分）→ 文字起こし（AI）→ 医学用語補正 → SOAP形式カルテ自動生成 → 医師が確認・編集 → 保存
```

### 4.5 配送

```
決済完了 → 発送待ち一覧 → CSV出力（ヤマト/ゆうパック）→ 追跡番号登録 → LINE発送通知
```

### 4.6 ステップ配信

```
トリガー発火（友だち追加/タグ/キーワード）→ シナリオにエンロール → 遅延判定 → メッセージ送信 → 次ステップ or 完了
```

## 5. セキュリティ

| 対策 | 概要 |
|------|------|
| 通信暗号化 | SSL/TLS（HTTPS）による全通信の暗号化 |
| データ暗号化 | AES-256-GCM による設定情報の暗号化保存 |
| 行レベルセキュリティ | テナントごとのデータ分離（RLS） |
| CSRF保護 | Double Submit Cookieパターン |
| レート制限 | ログイン試行回数の制限 |
| CSP | Content Security Policy による XSS 対策 |
| 監査ログ | 管理操作の全履歴記録 |
| セッション管理 | サーバー側セッション管理（有効期限・同時セッション制限） |
| ログインアラート | 新しいIPからのログイン時にメール通知 |
| Webhook署名検証 | 外部サービスからのWebhookの署名検証 |

- 内科
